# Sitemap Research

This context describes the language used to monitor competitor sitemaps and turn newly found URLs into keyword research opportunities.

## Language

**Research Keyword**:
A keyword inferred from a monitored URL and used as the query for SERP research. The URL is the primary source; page metadata, title, and h1 are fallback sources when the URL is unclear.
_Avoid_: keyword, main keyword, SEO keyword

**Monitored Site**:
A competitor or reference site whose sitemap is tracked for new URLs.
_Avoid_: website, source site

**Sitemap URL**:
The sitemap endpoint configured for a monitored site.
_Avoid_: sitemap link, sitemap address

**Opportunity Candidate**:
A research finding produced from a research keyword when SERP results suggest a potentially useful competitor or reference site. It is reviewed by a human before it becomes a monitored site.
_Avoid_: opportunity, candidate site, auto-added site

**SERP Review**:
The human-in-the-loop process of searching a research keyword and inspecting the top search results for newly registered or low-authority domains.
_Avoid_: SERP automation, Google scraping

**SiteData Signal**:
Domain age and DR information shown on a Google results page by the user's SiteData browser extension during SERP review.
_Avoid_: DR API, domain API

**Agent Review Run**:
A Codex-guided SERP review session where the agent reads today's sitemap diff, derives research keywords, searches them in the user's Chrome browser, and records opportunity candidates from visible SiteData signals.
_Avoid_: research script, crawler, scraper
